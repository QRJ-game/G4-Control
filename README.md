G4 Control v0.9.2 Public Beta
================================

Расширенный локальный WebUI для мобильного роутера G4 PRO.

Это не полная прошивка и не образ разделов. Установщик сохраняет заводской
интерфейс, внедряет G4 Control в штатный /index.html через bind-mount и
добавляет небольшой локальный ARM helper для функций, которых нет в обычном
WebUI.

АВТОРЫ
------
sleo441 и ChatGPT.

ПРОВЕРЕННАЯ СОВМЕСТИМОСТЬ
-------------------------
Устройство: G4 PRO / 4G CPE WiFi6 Powerbank
Hardware: G4_V1.1
Firmware: G4_B8F_1G030X_T3_D7012
Kernel: Linux 3.4.110-rt140, ARMv7
USB: ZTE VID_19D2, ADB + RNDIS

На других аппаратных и программных ревизиях работоспособность не
гарантируется. Установщик выполняет предварительные проверки, но они не могут
охватить все возможные варианты китайских ревизий устройства.

ОСНОВНЫЕ ВОЗМОЖНОСТИ
---------------------
- современный тёмный адаптивный WebUI;
- показатели LTE: диапазон, EARFCN, PCI, Cell ID, eNodeB, RSRP/RSRQ/SINR;
- выбор LTE-диапазонов и Cell Lock;
- изменение TTL и IMEI;
- USB Switch;
- список клиентов;
- настройки локальной Wi-Fi-точки;
- корректное чтение и сохранение Wi-Fi-пароля через штатный Base64 API;
- WISP: подключение к другой Wi-Fi-сети по SSID и паролю;
- временное отключение локального Wi-Fi до следующей перезагрузки;
- публичный режим: запрет управления роутером для Wi-Fi-клиентов до следующей
  перезагрузки;
- экспериментальное назначение Ethernet-порта LAN/WAN/AUTO;
- видимые LTE-соты из штатного ZLBS-потока;
- монитор CPU, RAM, swap/zram, разделов, uptime, helper, батареи и датчиков;
- ручное обновление системных показателей или опциональное обновление раз в 2 с;
- автоматический контроль runtime helper раз в минуту;
- автоматический перезапуск helper при аварийном завершении;
- безопасный возврат к заводскому UI и полный откат;
- аварийное восстановление Wi-Fi через RESTORE_WIFI.cmd.

HELPER И АВТОПЕРЕЗАПУСК
-----------------------
Runtime helper слушает только локальный TCP-порт 18081 и предоставляет
ограниченный набор встроенных операций. Произвольное выполнение shell-команд
через его HTTP API не реализовано.

Отдельный supervisor проверяет helper раз в минуту. Если helper завершился,
supervisor запускает его снова. Автоперезапуск включён по умолчанию и
сохраняется между перезагрузками. Тумблер находится во вкладке:

  Система -> Runtime helper -> Автоперезапуск

Если автоперезапуск выключен и helper остановлен вручную, вернуть его можно:

  adb shell "/mnt/userdata/g4ui/g4-boot.sh restart"

ТРЕБОВАНИЯ
----------
- Windows 10/11;
- распакованный архив;
- adb.exe в C:\platform-tools\adb.exe, в PATH либо в папке platform-tools
  рядом с установщиком;
- один подключённый по USB роутер;
- включённый USB Switch / ADB.

В штатном WebUI пункт USB Switch обычно недоступен при входе с паролем admin.
На проверенной прошивке расширенный пароль: factoryAdmin.
На других ревизиях пароль может отличаться.

УСТАНОВКА И ОБНОВЛЕНИЕ
----------------------
1. Включите usb_switch в WebUI роутера по ссылке:
   http://192.168.0.1/index.html#telnetd

2. Распакуйте скачанный архив в отдельную папку.

3. Подключите роутер к компьютеру по USB.

4. Проверьте в терминале:

     adb devices

   Должно отображаться ровно одно устройство со статусом device.

5. Запустите:

     INSTALL_OR_UPDATE.cmd

6. Дождитесь:

     Injected G4 host: HTTP verification OK
     Runtime helper: HTTP API v0.9.2 is running on TCP 18081
     helper=running
     cleaner=running
     12/12 Done.

7. Откройте:

     http://192.168.0.1/index.html?g4=new&build=0902

8. Выполните Ctrl+F5.

Установщик можно запускать поверх предыдущей версии G4 Control.

ВОЗВРАТ И ВОССТАНОВЛЕНИЕ
------------------------
FACTORY_UI.cmd
  Отключает G4 Control и возвращает заводской интерфейс. Файлы установки
  сохраняются, поэтому G4 Control можно включить снова.

ENABLE_UI.cmd
  Повторно включает установленный G4 Control.

ROLLBACK.cmd
  Удаляет постоянный hook, восстанавливает исходный customer.sh и возвращает
  заводской интерфейс.

RESTORE_WIFI.cmd
  Аварийно восстанавливает локальную точку Wi-Fi, если пароль или
  hostapd.conf были повреждены. Запрашивает новый пароль скрытым вводом,
  кодирует его в Base64, проверяет WPAPSK1_encode и отправляет штатную команду
  включения Wi-Fi. Если SSID не появился, требуется полное выключение и
  повторное включение питания.

REPAIR_RUNTIME.cmd
  Переустанавливает только helper, boot service и runtime actions. Используется
  при диагностике, обычному пользователю обычно не требуется.

STATUS.cmd
  Показывает состояние bind-mount, helper, supervisor и свободного места.

DIAGNOSE_HELPER.cmd
  Проверяет /api/ping, /api/status, /api/cells, процессы, listeners и логи.

VERIFY_CURRENT_INSTALL.cmd
  Проверяет уже установленный UI и helper без повторной установки.

ИЗВЕСТНЫЕ ОГРАНИЧЕНИЯ
---------------------
1. Ethernet LAN/WAN пока не проверен на достаточном количестве устройств.
   Раздел считается экспериментальным.

2. WISP работает, но используется один радиомодуль 2,4 ГГц одновременно для
   приёма и раздачи. Скорость может быть во много раз ниже прямого подключения.

3. ZLBS показывает vendor-показатель «сигнал». Он не объявляется RSRP, пока
   его точная семантика не подтверждена.

4. Процент батареи читается из штатного vendor API battery_vol_percent.
   Количество делений берётся из battery_pers, напряжение — из sysfs.
   Значение температуры аккумулятора около 77 °C признано недостоверным и
   намеренно не используется.

5. G4 Control предоставляет полный локальный доступ без отдельной формы
   пароля. Для общественных мест используйте публичный режим.

6. Физический Reset и поведение на других ревизиях устройства требуют
   дополнительной проверки.

7. При смене LAN/WAN, сетевого адреса, Wi-Fi и Cell Lock можно потерять
   соединение с роутером. Держите USB/ADB доступным во время экспериментов.

8. Для WPA/WPA2 G4 Control принимает печатный ASCII-пароль длиной 8–63
   символа. Такое ограничение исключает повреждение hostapd.conf из-за
   неоднозначной кодировки vendor API.

9. Команда adb reboot на проверенном устройстве может не выполнять
   перезагрузку. При необходимости полностью выключите и снова включите
   питание роутера.

БЕЗОПАСНОСТЬ И ДАННЫЕ
---------------------
Архив не содержит:
- заводскую прошивку или дампы разделов;
- IMEI/IMSI/ICCID конкретного устройства;
- пароли Wi-Fi;
- сохранённые WISP-профили;
- Google Platform Tools.

Перед публикацией диагностик ВСЕГДА удаляйте IMEI, IMSI, ICCID, MAC-адреса,
SSID, пароли и серийный номер ADB.
